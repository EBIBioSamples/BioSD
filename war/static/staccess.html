SampleTab format for accessing Bioinfomaticians

Note: Specification of this file format is at an early stage. All details are subject to flux, depending on how useful they are in the face of real data.

In addition to the SampleTab information provided for biologists and for submitting bioinfomaticians, bioinformaticians reading SampleTab files should bear in mind the following instructions, guidelines, and notes.

The philosophy of the BioSample Database is to accept submissions from as wide a sources as possible, and to that end few meta-data requirements are placed on the submitter. However, in order to make these varied submissions useful a number of pre- and post-submission data curation steps are applied. These steps are typically mapping terms to ontologies (disease, phenotype, etc) and linking to other databases, but also inferring relationships between samples - such as when a cell line has been used for multiple investigations. In general, these curation steps are applied using publicly available software but in certain cases changes may be applied manually.

In particular, when accessing curated SampleTab files from BioSD the following should be expected:

  • Additional information
    • Several fields are added to submissions once that are in the BioSD.
    • MSI
      • Submission Tracking Identifier
        • Unique identifier for tracking this submission into and throughout the database
    • SCD
      • Sample Accession
        • Unique identifier for this sample throughout the database
        • Used for cross-references (e.g. Child Of, Derrived From, etc)
  • Ontology links
    • also known as Controlled Vocabulary
    • Described in more detail in SampleTab for submitting Bioinformaticians
      • Ontology term SCD columns will be followed by a Term Source REF column containing the Term Source of the ontology
      • Term Source values will be added to the Term Source Name in the MSI with associated URI and version.
    • Will be applied to as much data as possible, even apparently trivial characteristics such as "sex".
    • Will use a standard set of ontologies where possible, though others are also possible. In order of most preferred to least preferred the prefered ontologies are:
      • EFO
      • NEWT
    • Names will be used rather than identification numbers, for example in species names.
  • Samples from other databases
    • Some submissions are automatically gathered from other databases, such as DGVa, ENA SRA, AE, etc.
    • These can be identified by the following tags in the MSI
      • Database Name
      • Database URI
      • Database ID
    • These may not contain as much data or as well-curated data as samples submitted directly to BioSD. This depends on what information the receiving database requires and/or stores.
